Results Overview

First Step: Phoenix

Phoenix is a malicious domain classifier that collects passive DNS data and tries to classify the domains and cluster the eventual botnets. At the moment we are at an early development stage. Stefano Schiavoni (Google), was able to implement the classifier and run it against the data he collected during his master thesis period and to perform a few tests that confirmed the goodness of the model.

Once the data is crunched we have a list of domains along with some useful informations:

  • DGA or non-DGA label;
  • the eventual cluster it belongs to;
  • the history of its DNS requests and replies.

Given this huge and precious amount of data we aim at producing two main services:

  • a set of visualization tools to better investigate a single domain's data;
  • a set of APIs to retrieve the data Phoenix was able to produce.

We shall explain these two features in the next paragraphs.

Data, visualized

We want to produce a set of visualization tools aiming at making an analyst's work faster and easier. More specifically:

Maps
To instantaneously get an idea of where the botmasters had located their machines.
Timeline
To instantaneously get an idea of the time range a domain has been used for communication purposes.
Area Charts
To visualize the botnet communication activity toward a specific domain over time.

JSON friendly

We aim at making all the data displayed in every results page retrievable via HTML GET requests. The data can be downloaded both in JSON and .csv format. The APIs section explains thoroughly how to retrieve the required data.
